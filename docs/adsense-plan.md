# 낙찰노트 애드센스 준비 기획안

## 1. 목표

낙찰노트를 단순 경매 분석 웹앱이 아니라, 경매 입찰을 준비하는 사람이 공부하고 계산하고 기록할 수 있는 사이트로 확장한다.

방향은 다음과 같다.

```txt
경매 입문/실전 콘텐츠
+ 입찰 검토 도구
+ 입찰가/필요현금/세후수익 계산
+ 필수 문서 확인 체크리스트
+ 정책/면책/문의 페이지
```

현재 서비스가 도구 중심이라면, 애드센스 승인용 구조는 다음처럼 바꾼다.

```txt
지금: 경매 물건 검색/분석 도구
목표: 경매 공부 콘텐츠 + 경매 검토 도구 + 참고 계산기
```

## 2. 사이트 포지셔닝

사이트명은 현재처럼 `낙찰노트`를 유지한다.

포지션은 다음으로 잡는다.

```txt
초보 경매 투자자를 위한 입찰 전 검토 노트
```

서비스 설명 문장:

```txt
낙찰노트는 법원경매 물건을 검색하고, 시세·권리·리스크·입찰가·필요현금·세후수익을 정리해 입찰 전 판단을 돕는 검토 도구입니다.
```

단, 분석 결과가 확정 판단처럼 보이면 안 된다. 모든 계산과 판단은 참고용으로 표시한다.

## 3. 사이트 구조

```txt
/
├─ 홈
├─ 경매 도구
│  ├─ 물건 검색
│  ├─ 여러 사건 일괄조회
│  ├─ 매각기일 추천
│  ├─ 저장 후보 TOP 5
│  ├─ 온비드 공매
│  └─ 입찰가 산정
├─ 경매 가이드
│  ├─ 입찰 전 체크리스트
│  ├─ 매각물건명세서 보는 법
│  ├─ 현황조사보고서 보는 법
│  ├─ 감정평가서 보는 법
│  ├─ 말소기준권리 기초
│  ├─ 임차인 대항력 기초
│  ├─ 입찰보증금과 잔금 준비
│  ├─ 대출과 필요현금 계산
│  ├─ 세후수익 계산 예시
│  └─ 법원경매와 온비드 차이
├─ 서비스 소개
├─ 이용 안내
├─ 개인정보처리방침
├─ 면책 고지
└─ 문의하기
```

## 4. 우선 추가한 정적 페이지

첫 단계로 아래 정적 페이지를 추가한다.

```txt
public/about.html
public/guide.html
public/privacy.html
public/disclaimer.html
public/contact.html
```

이 페이지들은 도구 코어 JS에 의존하지 않는다. 애드센스 대비 기본 페이지와 콘텐츠 허브 역할만 한다.

## 5. 우선 작성할 콘텐츠 주제

최소 10개, 권장 15~20개까지 확장한다.

초기 5개 주제:

```txt
1. 경매 입찰 전 반드시 확인해야 할 7가지
2. 매각물건명세서 보는 법
3. 말소기준권리 기초
4. 필요현금 계산법
5. 입찰가 산정 예시: 취득비용부터 세후수익까지
```

확장 주제:

```txt
현황조사보고서 보는 법
감정평가서 보는 법
임차인 대항력 기초
선순위 임차인이 위험한 이유
유치권·법정지상권 확인 포인트
입찰보증금과 잔금 준비
법원경매와 온비드 공매 차이
저장 후보 TOP 5 활용법
최종 판단 메모 작성법
```

## 6. 광고 배치 원칙

초기에는 도구 화면에 광고를 붙이지 않는다.

우선 콘텐츠 글 중심으로만 적용한다.

허용 후보:

```txt
글 상단 아래
글 중간 1회
글 하단
데스크톱 사이드 영역
```

피해야 할 위치:

```txt
조회 버튼 주변
입찰가 입력창 주변
다음 단계 버튼 주변
저장 후보 버튼 주변
문서 열기 버튼 주변
검색 결과 카드 사이
```

낙찰노트는 버튼과 입력창이 많은 도구형 화면이므로 광고가 버튼처럼 보이거나 실수 클릭을 유도하면 안 된다.

## 7. 면책 원칙

모든 도구 및 가이드 페이지 하단에 아래 원칙을 유지한다.

```txt
본 서비스의 분석·계산 결과는 참고용입니다.
입찰 전 법원경매정보, 매각물건명세서, 현황조사보고서, 감정평가서, 등기부등본 및 전문가 검토를 통해 최종 확인해야 합니다.
```

대출 가능 여부, 세금, 권리관계, 명도 가능성을 확정적으로 단정하지 않는다.

## 8. 6/11 전 작업

Codex 사용 전까지는 위험한 코어 기능 수정 대신 아래 작업을 쌓는다.

```txt

3. 입찰가 계산표 요구사항 정리
4. 전체 코드 전수 리뷰표 템플릿 작성
5. API 계약표 템플릿 작성
6. storage key 점검표 작성
7. OWASP LLM Top 10 기준 점검표 작성
```
