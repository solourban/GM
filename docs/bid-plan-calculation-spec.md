# 입찰가 산정 탭 계산표 요구사항

입찰가 산정 탭은 사용자가 입찰 전 필요 현금, 예상 세금, 세후수익, 수익률을 참고용으로 계산할 수 있게 구성합니다. 본 문서는 구현 전에 계산 항목과 방향을 고정하기 위한 초안입니다.

## 입력 및 계산 항목

| 항목 | 설명 |
| --- | --- |
| 감정가 | 감정평가 기준 금액 |
| 최저가 | 현재 회차 최저매각가격 |
| 입찰가 | 사용자가 검토하는 예정 입찰 금액 |
| 취득세 + 지방세 | 취득 단계 세금 |
| 기타 등기비용 | 등기 관련 부대비용 |
| 법무비 | 법무사 또는 관련 처리 비용 |
| 미납관리비 | 인수 또는 협의 가능성이 있는 관리비 |
| 수리비 | 예상 보수 비용 |
| 명도비 | 점유 이전 및 협의 관련 예상 비용 |
| 대출금액 | 적용할 대출 원금 |
| 대출비율 | 입찰가 또는 기준금액 대비 대출 비율 |
| 감정가 60% 기준 대출 | 감정가 기준 보수적 대출 가능액 참고값 |
| 낙찰가 80% 기준 대출 | 입찰가 기준 대출 가능액 참고값 |
| 방공제 금액 | 지역/주택 유형별 차감 필요액 |
| 월 이자 | 대출금액과 금리 기준 월 이자 |
| 보유기간 이자 | 보유기간 전체 이자 |
| 중도상환수수료 | 매도 또는 상환 시 예상 수수료 |
| 총 취득비용 | 입찰가와 취득 관련 비용 합계 |
| 필요 현금 | 총 취득비용에서 대출금액을 뺀 금액 |
| 예상 매도가 | 사용자가 가정하는 매각 예상 금액 |
| 세전수익 | 예상 매도가에서 입찰가를 뺀 금액 |
| 매도 중개수수료 | 매도 단계 중개 비용 |
| 총 비용처리 금액 | 취득 관련 비용, 보유비용, 매도비용 합계 |
| 과세표준 | 세전수익에서 비용처리 금액을 뺀 금액 |
| 양도세 또는 소득세 | 과세표준 기준 예상 세금 |
| 양도세 지방세 | 양도세에 부가되는 지방세 |
| 총 비용 | 입찰가와 취득비용, 보유비용, 매도비용을 더한 세금 제외 비용 |
| 세후수익 | 예상 매도가에서 총 비용과 세금을 반영한 뒤 남는 금액 |
| 수익률 | 필요 현금 대비 세후수익 |

## 계산 방향

```txt
총 취득비용 =
입찰가
+ 취득세/지방세
+ 등기비용
+ 법무비
+ 미납관리비
+ 수리비
+ 명도비
+ 보유기간 이자
+ 중도상환수수료

필요 현금 =
총 취득비용 - 대출금액

세전수익 =
예상 매도가 - 입찰가

총 비용처리 금액 =
취득 관련 비용 + 보유비용 + 매도비용

과세표준 =
세전수익 - 비용처리 금액

세후수익 =
예상 매도가 - 총 비용 - 세금

수익률 =
세후수익 / 필요 현금
```

## 주의 문구

- 본 계산은 참고용입니다.
- 실제 대출 가능 여부는 금융기관 확인이 필요합니다.
- 세금은 세무 전문가 또는 관계 기관 확인이 필요합니다.
- 권리관계와 명도 가능성은 원본 서류와 현장 확인이 필요합니다.

## 현재 구현 대조

기준 파일: `public/app-v2-bid-plan.js`

| 요구 항목 | 현재 구현 상태 | 비고 |
| --- | --- | --- |
| 감정가 | 계산에 사용 | `reportNumbers().appraisedValue` |
| 최저가 | 계산에 사용 | `reportNumbers().minBid` |
| 입찰가 | 입력/표시 | `plannedBid`, `v2PlannedBidInput` |
| 취득세 + 지방세 | 입력 세율로 계산 | `acquisitionTaxRate`, `acquisitionTax` |
| 기타 등기비용 | 입력 | `registryCost` |
| 법무비 | 입력 | `legalFee` |
| 미납관리비 | 입력 | `unpaidManagementFee` |
| 수리비 | 입력 | `repairCost` |
| 명도비 | 입력 | `evictionCost` |
| 대출금액 | 입력 또는 자동 참고값 | `loanAmount` |
| 대출비율 | 감정가 기준/낙찰가 기준 입력 | `appraisalLoanRate`, `bidLoanRate` |
| 감정가 60% 기준 대출 | 계산됨 | `appraisalLoanCap`, 현재 별도 표시 없음 |
| 낙찰가 80% 기준 대출 | 계산됨 | `bidLoanCap`, 현재 별도 표시 없음 |
| 방공제 금액 | 입력/계산 반영 | `roomDeduction` |
| 월 이자 | 계산/표시 | `monthlyInterest` |
| 보유기간 이자 | 계산/표시 | `holdingInterest` |
| 중도상환수수료 | 입력 세율로 계산 | `prepaymentPenaltyRate`, `prepaymentPenalty`, 현재 별도 표시 없음 |
| 총 취득비용 | 계산/표시 | 입찰가 + 취득/보유 비용 |
| 필요 현금 | 계산/표시 | `totalBurden - loanAmount` |
| 예상 매도가 | 입력 | `expectedSalePrice` |
| 세전수익 | 계산됨 | `preTaxProfit`, 현재 별도 표시 없음 |
| 매도 중개수수료 | 입력 | `sellBrokerageFee` |
| 총 비용처리 금액 | 계산됨 | `deductibleCost`, 현재 별도 표시 없음 |
| 과세표준 | 계산/표시 | `taxableBase` |
| 양도세 또는 소득세 | 입력 세율로 계산 | `incomeTax`, 현재 총세금으로 합산 표시 |
| 양도세 지방세 | 입력 세율로 계산 | `localIncomeTax`, 현재 총세금으로 합산 표시 |
| 총 비용 | 계산/표시 일부 | `totalCost`, 현재 `입찰가+인수 포함 부담`, `총 취득비용`, `세금 참고액`으로 분산 표시 |
| 세후수익 | 계산/표시 | `afterTaxProfit` |
| 수익률 | 계산/표시 | `roi` |

## 구현 보완 후보

- 감정가 60% 기준 대출, 낙찰가 80% 기준 대출, 방공제 차감 전후를 한 줄로 표시한다.
- 세전수익, 총 비용처리 금액, 양도세, 지방세를 별도 행으로 표시한다.
- `totalCost`의 의미를 UI에서 "세금 포함 총 비용" 또는 "세금 제외 총 비용"으로 명확히 나눈다.
- 입찰가 산정 카드에 "본 계산은 참고용" 문구는 이미 있으나, 권리관계/명도 확인 문구도 함께 노출하는지 확인한다.
